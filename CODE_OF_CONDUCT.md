# Code of Conduct

## Purpose

This code of conduct establishes the professional standards expected of all contributors to the Shine Harvest engineering program. It applies to all interactions within this GitHub organization, including pull requests, issues, discussions, and review comments.

## Standards

### Professional Conduct

- Communicate with respect and precision
- Provide constructive, evidence-based feedback in reviews
- Acknowledge the work and expertise of other teams
- Keep discussions focused on engineering substance

### Technical Integrity

- Technical disagreements are resolved through the architecture review process, not through unilateral action
- Safety-critical decisions require documented rationale — not opinion or authority alone
- Claims about system behavior must reference traceable evidence
- State terms must be used as defined in the program state dictionary

### Confidentiality

- All repository content is proprietary and confidential
- Do not share program artifacts outside the organization without explicit authorization
- Operationally sensitive information (site locations, infrastructure details, pricing contracts) requires special handling per the security policies

### Safety Culture

- Raise safety concerns openly — they are never penalized
- Safety-critical changes require the documented review process, regardless of schedule pressure
- "It works" is not a substitute for "it is verified"

## Violations

Report conduct violations to the `exec-admin` team. Reports are handled confidentially. Violations may result in access revocation.

## Scope

This code of conduct applies to all Shine Harvest GitHub organization activity. It supplements, but does not replace, any applicable organizational policies.
